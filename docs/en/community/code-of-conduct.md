# Code of conduct

The `piighost` community welcomes anyone sharing the goal of making PII protection accessible and composable. This page summarises the behaviour expectations. For the authoritative version, see [`CODE_OF_CONDUCT.md`](https://github.com/Athroniaeth/piighost/blob/master/CODE_OF_CONDUCT.md) at the repository root.

## Scope

This code applies to every space managed by the project:

- GitHub issues, pull requests and discussions.
- Community meet-ups and events.
- Any other forum related to the project.

## Expected behaviour

### Stay welcoming and patient

Welcome newcomers regardless of their experience level. A question that looks basic to you may be someone's very first question.

### Disagree with respect

Technical choices are up for debate. Critique ideas, not people. Back arguments with examples and alternatives rather than value judgements.

### Report what goes wrong

If you witness or experience inappropriate behaviour in a project space, reach out to the maintainers via [private email through their GitHub profile](https://github.com/Athroniaeth) or through a [GitHub security advisory](https://github.com/Athroniaeth/piighost/security/advisories/new) if the matter is security-related.

## Enforcement

Maintainers may remove, edit or reject contributions that do not follow this code. Repeat violations can result in temporary or permanent exclusion from project spaces.
